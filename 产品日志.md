## 项目概述

**Nbook.ai** 是一个美甲师预约管理系统，包含：
- **客户端**：客户可以预约标准服务或提交定制款报价请求
- **美甲师端**：美甲师可以管理预约、报价、排班设置和查看经营数据

**技术栈：**
- 后端：Flask (Python)
- 数据库：SQLite + SQLAlchemy ORM
- 前端：Bootstrap 5 + JavaScript
- 图表：Chart.js

### 2024 - 苹果日历视图

#### 8. 日历视图后端逻辑

**文件：** `app/routes/dashboard.py` (overview 路由)

**修改内容：**
- 添加视图类型参数（月视图/周视图）
- 添加日期导航参数
- 根据视图类型获取预约数据
- 按日期组织预约数据（bookings_by_date）
- 计算日历网格日期（calendar_dates, week_dates）
- 计算导航日期（prev_nav_date, next_nav_date）

**代码变更：**
```python
view_type = request.args.get("view", "month")  # month 或 week
view_date_str = request.args.get("date", today.isoformat())

if view_type == "week":
    # 周视图：获取当前周的所有预约
    week_start_view = view_date - timedelta(days=view_date.weekday())
    week_end_view = week_start_view + timedelta(days=6)
    calendar_bookings = Booking.query.filter_by(...).filter(
        Booking.booking_date.between(week_start_view, week_end_view)
    ).all()
else:
    # 月视图：获取当前月的所有预约
    month_start = view_date.replace(day=1)
    # ... 计算月末日期
    calendar_bookings = Booking.query.filter_by(...).filter(
        Booking.booking_date.between(month_start, month_end)
    ).all()

# 按日期组织预约数据
bookings_by_date = {}
for booking in calendar_bookings:
    date_key = booking.booking_date.isoformat()
    if date_key not in bookings_by_date:
        bookings_by_date[date_key] = []
    bookings_by_date[date_key].append(booking)
```

## 开发笔记

### 关键决策

1. **排班系统设计**
   - 选择支持多时间段而非单时间段，提高灵活性
   - 使用 `day_of_week` (0-6) 表示周一到周日

2. **时间计算**
   - 使用 30 分钟为最小时间单位（slot）
   - 卸甲和建构各增加 30 分钟，简单直观

3. **图片存储**
   - 使用 UUID 生成文件名，避免冲突
   - 存储在 `static/uploads/` 目录
   - 数据库存储逗号分隔的文件名

4. **日历视图**
   - 选择月视图和周视图两种模式
   - 事件块颜色区分不同类型预约
   - 支持点击跳转到详情页

### 已知问题

1. **时区处理**
   - 当前使用 UTC 时间，可能需要根据实际需求调整

2. **图片管理**
   - 未实现图片删除功能
   - 未实现图片压缩

3. **预约冲突**
   - 当前只检测同一天的冲突，未跨天检测
